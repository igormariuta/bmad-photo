- (version) EXIF parsing library: ExifReader 4.41.0 (verified via npm registry, last published 2026-06-08). Chosen over exifr 7.1.3 (more popular but unpublished since 2021-08, no documented HEIC/HEIF support -- Lazy Cam is an iOS camera app and iPhone defaults to HEIC capture, so this is a real risk). ExifReader explicitly supports HEIC/HEIF/JPEG/PNG/TIFF/WebP/AVIF and offers length:'auto' partial-byte reads (<1ms typical phone JPEG).
- (decision) Ingest parsing runs inside a Web Worker, not chunked main-thread. Main thread posts File/ArrayBuffer per photo (or a batch) to the worker; worker parses via ExifReader and posts back normalized records + progress ticks feeding the Ingest-progress StatBar ('Parsing N/100'). Guarantees main-thread responsiveness regardless of individual file size/slowness, satisfying the PRD NFR (parse off main thread or chunked).
- (version) State management: Zustand 5.0.14 (verified via npm registry). One canonical store holds the full Ingested photo list (each tagged readable/unreadable + normalized EXIF fields). Two independent selectors derive from it: useReadablePhotos() feeds Insights unconditionally (no access to filter state); useFacetFilters()+useFilteredPhotos() feed Browse only. This selector-level separation is the enforceable invariant preventing Insights and Browse from silently re-converging or diverging in how they read the photo set.
- (decision) Hosting: Vercel for both apps (Gallery SPA via Vite build, Landing via Astro SSG build). Matches the sister repo lazy-blog-front's existing infra (Vercel Analytics + Speed Insights already in use), keeps ops consistent across the team's products. Turborepo remote caching on Vercel is a natural fit but not committed as its own AD yet.
- (direction) Proposed (pending user confirmation): paradigm named 'layered monorepo with feature-sliced apps' -- one-way dependency apps/* -> packages/ui -> packages/theme (theme has zero deps); packages/eslint-config carries the no-arbitrary-value lint (FR-4) applied to both apps + packages/ui; Gallery internally organized by feature (ingest/, insights/, browse/, photo-detail/) not by file-type.
